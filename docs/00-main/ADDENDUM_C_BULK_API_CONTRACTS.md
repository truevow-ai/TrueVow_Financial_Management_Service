# ADDENDUM C — "Excel Grid" Bulk API Contracts (Final)

**Date:** January 25, 2026  
**Status:** Required for Grid Functionality

To make the grid truly usable, bulk endpoints must follow these exact rules.

---

## C1) Bulk Upsert Rules

- Accept up to 2,000 rows per request
- Each row must include `client_row_id` (uuid generated by UI)
- Server returns:
  - canonical list of lines (with real `line_id`)
  - `row_version` increment
  - per-row validation errors (soft errors allowed in drafts)

---

## C2) Endpoint Shapes

### Journal lines:
- `POST /api/v1/fm/journal-entries/{je_id}/lines:bulkUpsert`

### AP bill lines:
- `POST /api/v1/fm/ap/bills/{bill_id}/lines:bulkUpsert`

### Payroll adjustments:
- `POST /api/v1/fm/payroll/runs/{run_id}/adjustments:bulkUpsert`

---

## C3) Payload

```json
{
  "row_version": 12,
  "rows": [
    {
      "client_row_id": "uuid",
      "line_id": null,
      "op": "UPSERT",
      "data": {
        "account_id": "uuid",
        "debit": "100.00",
        "credit": "0.00",
        "description": "Office supplies",
        "dimensions": {
          "cost_center": "uuid",
          "department": "uuid",
          "location": "uuid"
        },
        "memo": "Monthly office supplies"
      }
    },
    {
      "client_row_id": "uuid",
      "line_id": "uuid",
      "op": "DELETE"
    },
    {
      "client_row_id": "uuid",
      "line_id": "uuid",
      "op": "UPSERT",
      "data": {
        "account_id": "uuid",
        "debit": "0.00",
        "credit": "100.00",
        "description": "Updated description"
      }
    }
  ]
}
```

---

## C4) Response

```json
{
  "row_version": 13,
  "rows": [
    {
      "client_row_id": "uuid",
      "line_id": "uuid",
      "status": "OK",
      "errors": []
    },
    {
      "client_row_id": "uuid",
      "line_id": "uuid",
      "status": "ERROR",
      "errors": [
        {
          "field": "cost_center",
          "code": "REQUIRED",
          "message": "Missing cost center"
        }
      ]
    }
  ],
  "totals": {
    "debit": "100.00",
    "credit": "100.00",
    "balance": "0.00"
  }
}
```

---

## C5) Error Handling

- Soft errors (missing dimensions, invalid account) are allowed in DRAFT
- Hard errors (unbalanced, invalid period) block posting
- Return errors per row so UI can highlight specific cells
